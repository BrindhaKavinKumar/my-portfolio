This Portfolio is a DevOps Project

Not just a static page — this portfolio itself is deployed using real DevOps practices. Every commit triggers a full pipeline.

🚀 Deployment Pipeline

  • 1
    Code Push → GitHub PR triggers GitHub Actions workflow automatically
  • 2
    Build & Test → Lighthouse CI + HTML validation + broken link checks
  • 3
    Docker Build → Containerize with Nginx, push image to ECR with semantic tags
  • 4
    Terraform Apply → S3 + CloudFront provisioned/updated via IaC
  • 5
    Deploy → Deploy to S3, invalidate CloudFront cache, smoke test
  • 6
    Monitor → CloudWatch + UptimeRobot alerting on uptime and performance

⚙️ GitHub Actions Pipeline (deploy.yml)

name: Deploy Portfolio on: push: branches: [ main ] jobs: build-and-deploy: runs-on: ubuntu-latest steps: # 1. Checkout code - uses: actions/checkout@v3 # 2. Validate HTML - name: Lint HTML run: npx html-validate src/ # 3. Build Docker image - name: Build & Push to ECR run: | aws ecr get-login-password | docker login docker build -t portfolio:$GITHUB_SHA . docker push $ECR_REPO:$GITHUB_SHA # 4. Terraform apply - name: Deploy Infrastructure run: | cd terraform && terraform init terraform apply -auto-approve # 5. Deploy to S3 - name: Sync to S3 run: aws s3 sync src/ s3://$BUCKET --delete # 6. Invalidate CloudFront - name: Cache Invalidation run: aws cloudfront create-invalidation --paths "/*"

🏗️ Suggested Tech Stack

Hosting: AWS S3 + CloudFront (or Vercel/Netlify for simplicity)
Framework: Next.js (static export) or plain HTML/CSS/JS
CI/CD: GitHub Actions (free tier sufficient)
IaC: Terraform for S3/CloudFront provisioning
Container: Nginx Docker image (multi-stage build)
DNS: Route 53 with ACM SSL certificate
Monitoring: UptimeRobot (free) + CloudWatch alarms
State: Terraform remote state in S3 + DynamoDB lock
